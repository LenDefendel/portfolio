# Инструкции для агента

Этот репозиторий - дизайнерское портфолио на Vue 3, TypeScript и Vite. Проект показывает работы по организациям/местам работы: каждая организация является отдельным разделом меню и отдельной страницей с проектами, изображениями, описаниями и тегами.

## Главная идея

Пользователь должен уметь:

- открыть общую галерею всех работ;
- кликнуть по изображению проекта;
- попасть на страницу нужной организации;
- автоматически проскроллиться к конкретному изображению выбранного проекта;
- открыть изображение в полном размере;
- переключить светлую и темную тему.

Разделы "Обо мне", "Навыки" и "Контакты" второстепенные. Не усложняй их без прямой просьбы.

## Технологии

- Vue 3
- TypeScript
- Vue Router
- Vite
- Yarn
- Material Symbols
- Sharp для получения размеров изображений

## Важные файлы

- `src/data/portfolio.ts` - основной источник контента: профиль, организации, проекты, навыки.
- `src/data/generatedImages.ts` - автогенерируемые массивы изображений и размеров. Не редактируй вручную без необходимости.
- `scripts/update-image-dimensions.ts` - генерирует `generatedImages.ts`.
- `src/router/index.ts` - маршруты, редиректы и плавный скролл по hash-якорям.
- `src/components/SidebarMenu.vue` - боковое меню и мобильное меню.
- `src/components/ProjectsGrid.vue` - общая галерея, страницы организаций, галереи проектов и lightbox.
- `src/components/ProjectCard.vue` - карточка изображения в общей галерее.
- `src/composables/useTheme.ts` и `src/components/ThemeToggle.vue` - переключение темы.
- `App.vue` - общий layout и CSS-переменные тем.

## Контентная модель

Организации описываются в `portfolio.workplaces`.

```ts
{
  id: string
  name: string
  role: string
  period: string
  icon: string
}
```

Проекты описываются в `portfolio.projects`.

```ts
{
  id: string
  title: string
  description: string
  image: ProjectImage
  workplaceId: string
  tags: string[]
  link?: string
  images?: ProjectImage[]
}
```

`workplaceId` проекта должен совпадать с `id` одной из организаций. Проекты без существующей организации не попадают в общую галерею.

## Навигация

Приложение использует `createWebHashHistory('/portfolio/')`.

Основные маршруты:

- `/` - редирект на первую организацию.
- `/workplace/all` - общая галерея всех изображений.
- `/workplace/:id` - страница организации.
- `/about`, `/skills`, `/contact` - второстепенные разделы.
- `/project/:id` - редирект на страницу организации с якорем первого изображения проекта.

Карточка в общей галерее должна вести по схеме:

```text
/workplace/{workplaceId}#{projectId}-image-{imageIndex}
```

Не ломай эту схему: она обеспечивает переход из общей галереи к конкретному изображению на странице организации.

## Изображения

Изображения лежат в `src/assets`.

Сейчас используются две основные группы:

- `src/assets/freelance3d`
- `src/assets/wallpaper`

После добавления, удаления или замены изображений запускай:

```sh
yarn update-image-dimensions
```

Это обновит `src/data/generatedImages.ts`, чтобы у изображений были корректные `width` и `height`.

## Темы

Темная и светлая темы задаются CSS-переменными в `App.vue`. Выбор темы хранится в `localStorage` по ключу `portfolio-theme`.

При изменениях темы проверь:

- начальную установку темы в `index.html`;
- переключение в `useTheme.ts`;
- переменные `.dark` и `.light` в `App.vue`;
- читаемость текста и границ в обеих темах.

## UI-правила

- Портфолио должно оставаться визуально чистым и ориентированным на изображения.
- Не превращай приложение в лендинг: первый экран и меню должны вести к работам.
- Не добавляй лишние описательные блоки и инструкции внутри интерфейса.
- Сохраняй адаптивность: сайдбар на desktop, hamburger-меню на мобильных экранах.
- При изменениях карточек и галерей проверяй, что изображения не растягивают layout и не перекрывают текст.
- Для иконок используй Material Symbols, так как они уже подключены в `src/main.ts`.

## Команды

Установка зависимостей:

```sh
yarn
```

Разработка:

```sh
yarn dev
```

Сборка и type-check:

```sh
yarn build
```

Линтинг:

```sh
yarn lint
```

Обновление размеров изображений:

```sh
yarn update-image-dimensions
```

## Git

- Когда пользователь просит переименовать существующие коммиты, не проверяй наличие несохраненных или незакоммиченных изменений перед переименованием.

## Проверка после изменений

Минимально проверь:

- `yarn build` после изменений в TypeScript, Vue-компонентах, роутере или данных.
- `yarn update-image-dimensions` после изменений в `src/assets`.
- Переход из `/workplace/all` к конкретному изображению проекта.
- Открытие lightbox по клику на изображение на странице организации.
- Переключение темы.
- Мобильное меню при ширине меньше 768px.

Если менялся только Markdown, сборку можно не запускать.
